# AI工作流输出JSON自动打开业务弹窗 - 通用组件提取方案

## 1. 需求概述

工作流执行完成后，EndNode 输出 JSON（含 `ai_new_route`），需要自动弹出业务弹窗预填数据。

**使用场景**：
- 工作流管理页面（WorkflowRuntimeList.vue）— 当前已实现（内联代码）
- SCM智能助手对话界面（ChatPanel.vue）— 需要复用
- 未来更多页面

**核心变更**：把弹窗检测/加载逻辑从 WorkflowRuntimeList 提取为通用组件，采用**静态注册**方式（显式import所有AI弹窗，map映射路由）。任何页面只需 import 这个组件即可使用。

## 2. 调用链路分析

### 路径A：WorkflowRuntimeList（直接调用）

```
WorkflowRuntimeList.runWorkflow()
  → SSE onComplete(finalOutput)
    → this.$refs.aiDialogLoader.open(finalOutput)
      → AiBusinessDialogLoader.open(output)
        → stripMarkdownJson(output) → JSON.parse
        → 检测 ai_new_route → 在 AI_DIALOG_MAP 中查找组件
        → 渲染弹窗
```

### 路径B：ChatPanel（通过Vuex Store中转）

```
store.dispatch('chat/sendMessage') 或 dispatch('chat/executeWorkflowCommand')
  → SSE onComplete(finalContent)
    → commit('SET_PENDING_AI_DIALOG_OUTPUT', finalContent)
      → ChatPanel watch pendingAiDialogOutput
        → this.$refs.aiDialogLoader.open(val)
        → commit('chat/SET_PENDING_AI_DIALOG_OUTPUT', null)  // 消费后清空
          → AiBusinessDialogLoader.open(output)
            →（同路径A后续逻辑）
```

## 3. 关键发现

### Store命名空间

`chat` 模块注册在根级别（src/components/70_ai/index.js:60）：
```js
store.registerModule('chat', aiStoreModules.modules.chat)
```
正确的 commit 路径是 `chat/SET_PENDING_AI_DIALOG_OUTPUT`，不是 `ai/chat/...`。

### 双触发点

`sendMessage` 和 `executeWorkflowCommand` 的 `onComplete` 都需要触发弹窗检测。

### 弹窗组件接口

所有AI业务弹窗遵循统一接口：
- props: `visible`(Boolean), `aiData`(Object)
- events: `closeMeOk`, `closeMeCancel`

## 4. 按文件设计

### 4.1 新建：AiBusinessDialogLoader.vue

**位置**：`src/components/70_ai/components/common/AiBusinessDialogLoader.vue`

**职责**：接收工作流输出文本 → 解析JSON → 检测`ai_new_route` → 在静态映射表中查找组件 → 渲染弹窗

**静态注册表设计**：
```js
import PoProjectAiNew from '@/views/40_business/10_po/project/dialog/ai/new/index.vue'

// AI业务弹窗路由映射表（新增弹窗时在此添加import和映射）
const AI_DIALOG_MAP = {
  'src/views/40_business/10_po/project/dialog/ai/new/index.vue': PoProjectAiNew
}
```

**对外接口**：
- 方法：`open(output)` — 父组件通过 `$refs` 调用
- 事件：`@close` — 弹窗关闭时通知父组件

**核心逻辑**：
```js
open (output) {
  if (!output || typeof output !== 'string') return
  try {
    const jsonStr = this.stripMarkdownJson(output)
    const data = JSON.parse(jsonStr)
    if (!data.ai_new_route) return

    const component = AI_DIALOG_MAP[data.ai_new_route]
    if (!component) {
      console.warn('[AiBusinessDialogLoader] 未注册的弹窗路由:', data.ai_new_route)
      return
    }

    this.aiDialog = { visible: true, component, data }
  } catch (e) { /* 非JSON忽略 */ }
}
```

**模板**：
```html
<component
  v-if="aiDialog.visible && aiDialog.component"
  :is="aiDialog.component"
  :visible="aiDialog.visible"
  :ai-data="aiDialog.data"
  @closeMeOk="close"
  @closeMeCancel="close"
/>
```

## 6. 扩展方式

未来新增业务弹窗只需：
1. 在业务目录创建 `dialog/ai/new/index.vue`（遵循统一接口：props visible/aiData，events closeMeOk/closeMeCancel）
2. 在 `AiBusinessDialogLoader.vue` 中添加 import 和 AI_DIALOG_MAP 映射

通用组件的使用方（工作流管理、智能助手等）不需要改。

## 7. 向后兼容

- 非JSON输出 → `open()` 内 try/catch 静默忽略
- JSON无 `ai_new_route` → 不弹窗
- `ai_new_route` 未注册 → console.warn 提示，不弹窗
- 无工作流的普通对话 → `pendingAiDialogOutput` 不会被设置
- WorkflowRuntimeList 行为完全不变

## 8. KISS原则7问题回答

1. **真问题还是臆想？** — 真问题，ChatPanel需要复用弹窗逻辑
2. **有更简单的方法？** — 提取组件+静态注册是最简方案
3. **会破坏什么？** — 不会，纯重构+新增
4. **真的需要？** — 是的，两个场景需要共享逻辑
5. **过度设计了吗？** — 没有，~55行组件
6. **话题是否模糊？** — 不模糊，代码已存在只是提取
7. **学习了注意事项？** — 已确认：前端文件位置（70_ai/components/common）、store命名空间（chat/）、组件接口

## 9. 风险分析

| 风险 | 概率 | 影响 | 缓解 |
|------|------|------|------|
| Store命名空间写错 | 低 | 弹窗不触发 | 已确认是`chat/`而非`ai/chat/` |
| 弹窗路由key不匹配 | 低 | 弹窗不弹出 | 编译时可发现import错误，key来自知识库可控 |
| watch触发时组件未渲染 | 低 | ref为null | 使用$nextTick |
