```
用户操作 → 前端 ModelSettingsDialog.vue
         → api/model.js editModelConfig()

         → AiModelConfigController.editModelConfig()
         → AiModelConfigService.editModelConfig()
         → buildModelConfig()
         → AiModelConfigMapper.updateById()
         → MySQL ai_model_config 表
```

**删除链路**：
```
用户点删除 → ModelSettingsDialog.vue deleteModel()
           → api/model.js deleteModelConfig()
           → DELETE /api/v1/ai/model/config/delete/{id}

```

## 二、问题诊断与根因

### CRITICAL-1：API Key 覆盖问题

**根因**：
- 列表查询时，后端 `maskSkString()` 返回脱敏值 `"sk-AB**** xx"`
- 用户切换启用/禁用开关时，前端把列表中已脱敏的 apiKey 直接传回后端
- 后端 `buildModelConfig()` 未检测脱敏值，直接写入 DB
- 结果：真实 API Key 被 `"sk-AB**** xx"` 覆盖，模型调用失败

**位置**：
- 前端：`ModelSettingsDialog.vue` Line 442 `apiKey: item.apiKey`
- 后端：`AiModelConfigService.java` Line 110-112 `buildModelConfig()`

### CRITICAL-2：删除默认模型无保护

**根因**：
- `deleteModelConfig()` 只检查记录是否存在，直接删除
- 未查询 `ai_config` 表中 `DEFAULT_LLM_MODEL_ID` 等键
- 删除默认模型后，AI所有功能调用 `getDefaultModelConfig()` 时报 `"模型配置不存在"` 异常

**位置**：`AiModelConfigService.java` Line 188-196

### CRITICAL-3：DB model_type DEFAULT 'CHAT'

**根因**：
- 数据库字段 `model_type DEFAULT 'CHAT'`
- 代码中所有模型类型使用 `'LLM'`（`ModelTypeEnum.LLM = 'LLM'`）
- 如果代码路径未显式设置 modelType，DB 会写入 'CHAT'

**位置**：数据库 `ai_model_config` 表 DDL

### HIGH-1：maxTokens 默认值三处不一致

| 位置 | 值 |
|------|----|
| 数据库 ai_model_config.max_tokens DEFAULT | 2000 |
| 后端 validateAndSetAdvancedSettings() | 1024 |
| 前端 buildAdvSettingDTOList() | 4096 |

**统一目标**：4096（现代大模型的合理默认值）

### MEDIUM-2：列表搜索未包含 name 字段

**根因**：`selectListWithUserName()` SQL 搜索条件只有 `model_name` 和 `deployment_name`，用户看到的是 `name`（显示名称），按显示名搜索无结果。

## 四、方案设计（按文件）

### 后端修改

#### 文件1：`AiModelConfigService.java`

**改动1 - CRITICAL-1**：在 `buildModelConfig()` 中检测脱敏 API Key，跳过更新

```java
// 新增私有方法
private boolean isApiKeyMasked(String apiKey) {
    return apiKey != null && apiKey.contains("****");
}

// buildModelConfig() 中修改 apiKey 处理逻辑
if (aiModelConfigVo.getApiKey() != null && !isApiKeyMasked(aiModelConfigVo.getApiKey())) {
    aiModelConfig.setApiKey(aiModelConfigVo.getApiKey());
}
```

**改动2 - CRITICAL-2**：`deleteModelConfig()` 删除前检查是否为默认模型

```java
public void deleteModelConfig(Long id, String userId) {
    AiModelConfigEntity entity = aiModelConfigMapper.selectById(id);
    if (entity == null) {
        throw new RuntimeException("模型信息不存在");
    }
    // 检查是否为默认模型
    String idStr = id.toString();
    if (idStr.equals(aiConfigService.getConfigValue("DEFAULT_LLM_MODEL_ID"))
        || idStr.equals(aiConfigService.getConfigValue("DEFAULT_VISION_MODEL_ID"))
        || idStr.equals(aiConfigService.getConfigValue("DEFAULT_EMBEDDING_MODEL_ID"))) {
        throw new RuntimeException("该模型已设为默认模型，请先更换默认模型后再删除");
    }
    aiModelConfigMapper.deleteById(id);
    log.info("删除模型配置成功, id: {}", id);
}
```

**改动3 - HIGH-1**：`validateAndSetAdvancedSettings()` 中将 maxTokens 默认值改为 4096

```java
} else if (entity.getMaxTokens() == null) {
    entity.setMaxTokens(4096); // 从1024改为4096
}
```

#### 文件2：`AiModelConfigMapper.java`

**改动4 - MEDIUM-2**：`selectListWithUserName()` SQL 搜索条件新增 `name` 字段

```sql
AND (t1.model_name LIKE CONCAT('%', #{keyword}, '%')
     OR t1.deployment_name LIKE CONCAT('%', #{keyword}, '%')
     OR t1.name LIKE CONCAT('%', #{keyword}, '%'))
```

### 数据库修改

**改动5 - CRITICAL-3**：修改 model_type 字段默认值

```sql
ALTER TABLE ai_model_config MODIFY COLUMN model_type VARCHAR(20) NOT NULL DEFAULT 'LLM' COMMENT '模型类型：LLM/VISION/EMBEDDING';
```

### 前端修改

无需修改（CRITICAL-1在后端防御即可，前端 changeStatus 逻辑本身没有问题）
